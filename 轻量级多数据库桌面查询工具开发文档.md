# 轻量级多数据库桌面查询工具开发文档

## 一、项目概述

### 1.1 项目名称

轻量级多数据库桌面查询工具

可暂定名称：

```
DB Lite Desktop

```

---

### 1.2 项目定位

本工具是一个轻量级桌面端数据库连接与查询工具。

它不是 DBeaver、Navicat 这类大型数据库管理软件，而是一个小型、安全、够用的数据库查看工具。

核心目标是：

1. 支持连接多种数据库。
2. 可以查看数据库中的表。
3. 可以查看每张表的字段。
4. 可以编写 SQL 查询数据。
5. 默认只允许只读查询。
6. 不启动 Web 服务。
7. 不暴露 HTTP 接口。
8. 占用资源低，启动速度快。

---

### 1.3 使用场景

适用于以下场景：

1. 开发人员快速查看数据库表结构。
2. 运维人员临时查询数据库数据。
3. 数据分析人员执行简单 SQL。
4. 项目内部人员查看业务数据。
5. 替代大型数据库工具中的轻量查询功能。

---

## 二、核心原则

### 2.1 轻量原则

本工具必须满足：

```
启动快
占用低
界面简单
功能聚焦
不做复杂数据库管理

```

---

### 2.2 安全原则

安全设计必须满足：

```
不使用 Web 前端
不启动本地 HTTP 服务
不暴露端口
密码加密保存
默认只允许只读 SQL
禁止危险 SQL
禁止多语句执行
限制查询返回行数
设置查询超时时间

```

---

### 2.3 功能边界

第一版只做：

```
数据库连接
测试连接
查看表
查看字段
SQL 查询
查询结果展示
保存连接配置
密码加密保存
查询历史

```

第一版不做：

```
建表
删表
改表
修改字段
编辑数据
删除数据
导入导出
ER 图
数据同步
数据库备份恢复

```

## 三、技术选型

### 3.1 推荐技术方案

推荐使用：

```
C# WinForms
+
Provider 数据库适配器模式
+
SQLite 本地配置库
+
Windows DPAPI 密码加密

```

---

### 3.2 为什么选择 C# WinForms

选择 WinForms 的原因：

1. 适合 Windows 桌面端。
2. 开发速度快。
3. 程序比 Electron 更轻。
4. 不需要浏览器。
5. 不启动 Web 服务。
6. TreeView 适合展示数据库表。
7. DataGridView 适合展示字段和查询结果。
8. 很适合做轻量工具。

---

### 3.3 为什么不用 Web 前端

不推荐：

```
Vue
React
Electron
FastAPI + 浏览器页面
本地 Web 服务

```

原因：

1. 会增加整体体积。
2. 会引入前后端通信。
3. 需要启动端口。
4. 存在本地接口暴露风险。
5. 不符合轻量和安全定位。

## 十三、SQL 安全设计

### 13.1 允许的 SQL

默认只允许：

```
SELECT
SHOW
DESC
DESCRIBE
EXPLAIN
PRAGMA
WITH

```

说明：

1. SELECT 用于普通查询。
2. SHOW 用于 MySQL。
3. DESC / DESCRIBE 用于查看结构。
4. EXPLAIN 用于查看执行计划。
5. PRAGMA 用于 SQLite。
6. WITH 用于 CTE 查询。

---

### 13.2 禁止的 SQL

必须禁止：

```
INSERT
UPDATE
DELETE
DROP
ALTER
TRUNCATE
CREATE
REPLACE
GRANT
REVOKE
MERGE
EXEC
EXECUTE
CALL

```

---

### 13.3 禁止多语句

禁止这种 SQL：

```
SELECT * FROM users; DROP TABLE users;

```

第一版规则：

```
只允许一次执行一条 SQL

```

### 13.4 SQLGuard 实现

```
using System.Text.RegularExpressions;

public class SqlGuardService
{
    private static readonly string[] AllowedPrefixes =
    {
        "select",
        "show",
        "desc",
        "describe",
        "explain",
        "pragma",
        "with"
    };

    private static readonly string[] BlockedKeywords =
    {
        "insert",
        "update",
        "delete",
        "drop",
        "alter",
        "truncate",
        "create",
        "replace",
        "grant",
        "revoke",
        "merge",
        "exec",
        "execute",
        "call"
    };

    public bool IsReadonlySql(string sql)
    {
        if (string.IsNullOrWhiteSpace(sql))
        {
            return false;
        }

        var cleanSql = sql.Trim().ToLower();

        var statements = cleanSql.Split(';')
            .Where(x => !string.IsNullOrWhiteSpace(x))
            .ToList();

        if (statements.Count > 1)
        {
            return false;
        }

        var firstWord = cleanSql.Split(' ', '\n', '\r', '\t')[0];

        if (!AllowedPrefixes.Contains(firstWord))
        {
            return false;
        }

        foreach (var keyword in BlockedKeywords)
        {
            if (Regex.IsMatch(cleanSql, $@"\b{keyword}\b"))
            {
                return false;
            }
        }

        return true;
    }
}

```

### 14.2 密码加密服务

```
using System.Security.Cryptography;
using System.Text;

public class PasswordEncryptService
{
    public string Encrypt(string plainText)
    {
        if (string.IsNullOrEmpty(plainText))
        {
            return string.Empty;
        }

        var data = Encoding.UTF8.GetBytes(plainText);

        var encrypted = ProtectedData.Protect(
            data,
            null,
            DataProtectionScope.CurrentUser
        );

        return Convert.ToBase64String(encrypted);
    }

    public string Decrypt(string encryptedText)
    {
        if (string.IsNullOrEmpty(encryptedText))
        {
            return string.Empty;
        }

        var data = Convert.FromBase64String(encryptedText);

        var decrypted = ProtectedData.Unprotect(
            data,
            null,
            DataProtectionScope.CurrentUser
        );

        return Encoding.UTF8.GetString(decrypted);
    }
}

```

## 十五、MySQL Provider 示例

```
using MySqlConnector;
using MySqlConnector.MySqlClient;
using System.Data;

public class MySqlProvider : IDatabaseProvider
{
    private string BuildConnectionString(DbConnectionConfig config, string password)
    {
        return $"Server={config.Host};" +
               $"Port={config.Port};" +
               $"Database={config.DatabaseName};" +
               $"User ID={config.Username};" +
               $"Password={password};" +
               $"Charset=utf8mb4;" +
               $"Connection Timeout=10;" +
               $"Default Command Timeout=30;" +
               $"Pooling=false;";
    }

    public bool TestConnection(DbConnectionConfig config, string password)
    {
        using var conn = new MySqlConnection(BuildConnectionString(config, password));
        conn.Open();

        using var cmd = new MySqlCommand("SELECT 1", conn);

        return cmd.ExecuteScalar() != null;
    }

    public List<string> GetTables(DbConnectionConfig config, string password)
    {
        var tables = new List<string>();

        using var conn = new MySqlConnection(BuildConnectionString(config, password));
        conn.Open();

        using var cmd = new MySqlCommand("SHOW TABLES", conn);
        using var reader = cmd.ExecuteReader();

        while (reader.Read())
        {
            tables.Add(reader.GetString(0));
        }

        return tables;
    }

    public List<TableColumnInfo> GetColumns(
        DbConnectionConfig config,
        string password,
        string tableName)
    {
        var result = new List<TableColumnInfo>();

        using var conn = new MySqlConnection(BuildConnectionString(config, password));
        conn.Open();

        var sql = $"SHOW FULL COLUMNS FROM `{tableName}`";

        using var cmd = new MySqlCommand(sql, conn);
        using var reader = cmd.ExecuteReader();

        while (reader.Read())
        {
            result.Add(new TableColumnInfo
            {
                Name = reader["Field"]?.ToString(),
                Type = reader["Type"]?.ToString(),
                Nullable = reader["Null"]?.ToString(),
                Key = reader["Key"]?.ToString(),
                DefaultValue = reader["Default"]?.ToString(),
                Extra = reader["Extra"]?.ToString(),
                Comment = reader["Comment"]?.ToString()
            });
        }

        return result;
    }

    public DataTable ExecuteQuery(
        DbConnectionConfig config,
        string password,
        string sql,
        int maxRows = 1000)
    {
        var guard = new SqlGuardService();

        if (!guard.IsReadonlySql(sql))
        {
            throw new Exception("当前工具只允许执行只读 SQL");
        }

        using var conn = new MySqlConnection(BuildConnectionString(config, password));
        conn.Open();

        using var cmd = new MySqlCommand(sql, conn);
        cmd.CommandTimeout = 30;

        using var adapter = new MySqlDataAdapter(cmd);

        var table = new DataTable();

        adapter.Fill(0, maxRows, table);

        return table;
    }

    public string BuildPreviewSql(string tableName, int limit = 100)
    {
        return $"SELECT * FROM `{tableName}` LIMIT {limit};";
    }
}

```

## 二十、主窗口核心逻辑

### 20.1 加载表列表

```
private void LoadTables()
{
    var provider = DatabaseProviderFactory.Create(currentConfig.DbType);

    var password = string.Empty;

    if (currentConfig.DbType != "sqlite")
    {
        password = passwordEncryptService.Decrypt(currentConfig.PasswordEncrypted);
    }

    var tables = provider.GetTables(currentConfig, password);

    tableTree.Nodes.Clear();

    foreach (var table in tables)
    {
        tableTree.Nodes.Add(table);
    }
}

```

### 20.3 执行 SQL

```
private void ExecuteSql()
{
    var provider = DatabaseProviderFactory.Create(currentConfig.DbType);

    var password = string.Empty;

    if (currentConfig.DbType != "sqlite")
    {
        password = passwordEncryptService.Decrypt(currentConfig.PasswordEncrypted);
    }

    var start = DateTime.Now;

    try
    {
        var result = provider.ExecuteQuery(
            currentConfig,
            password,
            sqlTextBox.Text,
            1000
        );

        var duration = (DateTime.Now - start).TotalMilliseconds;

        resultGrid.DataSource = result;

        statusLabel.Text = $"查询成功，返回 {result.Rows.Count} 行，耗时 {duration} ms";
    }
    catch (Exception ex)
    {
        MessageBox.Show(ex.Message, "查询失败");
    }
}

```

## 二十一、性能设计

### 21.1 不预加载

连接成功后只加载表名。

不要一次性加载所有字段和数据。

正确流程：

```
连接数据库
  ↓
读取表名
  ↓
点击表
  ↓
读取字段
  ↓
点击查询
  ↓
读取数据

```

---

### 21.2 不使用连接池

第一版建议连接字符串设置：

```
Pooling=false

```

原因：

1. 工具轻量。
2. 不需要常驻大量连接。
3. 每次查询用完关闭。
4. 更容易控制资源占用。

---

### 21.3 查询结果限制

默认：

```
预览数据：100 行
普通查询：最多 1000 行
查询超时：30 秒
连接超时：10 秒

```

## 二十二、安全建议

### 22.1 数据库账户使用只读权限

这是最重要的安全措施。

即使工具限制了 SQL，也建议数据库账户本身只给只读权限。

MySQL 示例：

```
GRANT SELECT, SHOW VIEW ON shop_db.* TO 'readonly_user'@'%';

```

不要给：

```
INSERT
UPDATE
DELETE
DROP
ALTER
CREATE

```

---

### 22.2 程序只读限制不是最终安全边界

程序中的 SQLGuard 主要用于防误操作。

真正安全应该依赖：

```
数据库账号权限
+
程序 SQL 白名单
+
查询超时
+
返回行数限制

```
